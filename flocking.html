Drone flocking perception and navigation
Challenge
This project was my end-of-study project done for a company called MBDA. The goal is to build the drone perception
for flocking application. Flocking is a principle describing birds behaviour. Basically, the goal is to use both camera
perception and flocking behaviour to build a fleet of drone on a simulator that is able to move from A to B in coordination.
For this project, I deciced to split the project in 3 parts : the perception, the flocking and the implementation on the
simulator.
Part 1. Perception
Dataset
For the perception model, I am using the cityscapes dataset which contains, in the version I am using 5000 images from different cities (mainly in Germany), a segmentation file which contains the labelled image segmented and the JSON files with the image information (annotations, etc...).
Model
Here we want to perform semantic segmentation in order to understand the whole environment and not only some object.
The principle of semantic segmentation is to perform a classification for each pixel of the input image. In other words, we
have a defined number of classes and each of them are associated to a value or a color and the model has to convert a
classic RGB image as shown above to a mapped image called mask. We use the encoder-decoder architecture called UNet. Below you can see the architecture of the model.
Results
Of course, the result is not perfect or ideal. There are many reasons behind that, the first is that I am training this model on a really small dataset for image generation. Another reason is that I am not training it enough but for a longer training, it requires more GPUs or more time. I considered that this was enough for now and later on it will be interesting to re train it.
